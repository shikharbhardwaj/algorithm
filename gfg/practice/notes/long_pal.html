===========
Approaches
===========
1. Naive approach : O(n^3), try every combination of i, j such that i, j ϵ {1..n}, i < j and check if ar[i..j] is a palindrome.
2. Optimized approach :
  a. For each character in the string, compute the next positions it comes in - O(n)
  b. We now know that if the string began at this position, it must end at one of the other positions of this starting character (!)

[a a a a b b a a]

{
  a -> [0, 1, 2, 3, 6, 7],
  b -> [4, 5]
}

Can improve complexity a bit, but still not much better in the worst case.

3. What is a palindrome?

For odd length, the middle can be anything, we don't care, the rest is as it is for even length, the two parts must match.
ie. at each position in the string, check if this is the center of a palindrome, or if it is the beginning of one of the halves.
